Episode 10 of "Raising an Agent" explains Amp's deep mode launch, the decision to retire the editor-sidebar workflow, and the broader claim that teams on the frontier need to reorganize around autonomous agents instead of human-paced assistant loops.

### Key Points Covered

- **Deep mode is a workflow, not just a model selector**: Amp's new deep mode uses GPT-5.2 Codex Medium for longer, well-scoped tasks that benefit from thorough research and delayed review. The hosts contrast it with smart mode for interactive work and rush mode for small fast changes [00:55]-[05:21].
- **Modes should change user expectations**: Quinn and Thorsten argue that putting a model in a drop-down is not enough. A product should help users feel when they are writing a short text-message-style prompt versus a longer letter to an agent that will go away and work [05:52]-[07:31].
- **The factory pattern becomes more natural as models run longer**: If an agent can work for 45-60 minutes without needing input, watching one sidebar thread stops making sense. The hosts expect more parallel agents, long-running tasks, and check-ins after completion [08:03]-[10:17].
- **Skills are accumulating operational knowledge**: Amp uses skills to encode release workflows, tmux testing details, G-Cloud log analysis, BigQuery usage, and other team-specific procedures so agents can navigate common tasks without repeated prompting [10:17]-[16:18].
- **Agent-optimized dev tooling can change human workflows**: The team discusses fast cached test commands and even accepting some regression in VS Code ergonomics when the tradeoff improves the feedback loops agents use most often [16:18]-[21:19].
- **Amp is killing the editor extension**: The hosts say the sidebar limits parallelism, keeps the human as the feedback loop, and would select for lagging users if Amp kept supporting it as a first-class path. They urge users to move to the CLI while the VS Code extension self-destructs [21:52]-[27:57].
- **Software product assumptions are melting**: Examples like AI-generated workout UIs, voice-guided workflows, and assistant-managed shopping lists show how software may become less tied to durable app interfaces and more tied to generated or conversational workflows [28:31]-[32:52].
- **Frontier companies need to keep re-earning their product**: Amp frames its lack of a traditional sales/marketing buildout, willingness to remove features, and appetite for risky changes as a strategic response to models that can overturn product assumptions every few months [33:25]-[43:49], [49:52]-[55:05].
- **Manual context management may be fading**: The episode ends by revisiting another old assumption: as models such as GPT-5.2 become better at researching after compaction, automatic handoff and hidden context-window management become more viable, though clear task boundaries still matter [43:49]-[49:19].
